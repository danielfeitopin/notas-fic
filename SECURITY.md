# Security Policy

Thank you for helping to keep this project and its users safe.

## Reporting a vulnerability

If you discover a security vulnerability, please **do not open a public issue**.

Instead, contact us privately at:

- Email: <danielfeitopin+github@protonmail.com>

When reporting, please include (if possible):

- A clear description of the issue and potential impact.
- Steps to reproduce the vulnerability.
- Any proof‑of‑concept code or screenshots.
- Affected version(s) and environment (OS, language/runtime, configuration).
- Whether the issue has been disclosed publicly anywhere else.
